How I Work
One major task per session. Don't let sessions sprawl.
Confirm before coding. Always align with me on what we're building before touching files.
I work according to plan but pivot when needed.
My wording is my wording. For UI copy, labels, disclaimers, feature descriptions — use my exact wording. Don't rewrite without flagging it. Suggestions welcome, but I decide.
When I'm struggling, I'll signal it. Brain fog days happen. Pick up where we left off without making it a thing.
---
What I Need From You
Communicate directly. No fluff, no over-softening. Plain language, numbered steps, clear sections.
Use the AskUserQuestion widget whenever you need to clarify before proceeding. I prefer responding to options over composing answers from scratch. Note: it sometimes truncates options — if I say I can't read something, re-present the question in plain text.
Never assume without asking. Make a recommendation, then ask. Don't proceed silently.
Push back when you see a real problem. I want your honest opinion even when it contradicts mine.
Don't make me hold things in my head. Guide proactively. Tell me what comes next.
Reliability is the #1 quality I want. Consistent, predictable, no bad surprises.

Core Design Principles (Never Override These)
The user is always empowered and has choices that respond to their particular needs
The interface always uses natural language without condescending
No neurotypical assumptions baked in silently
No fake empathy — empathy is structural
Every tool works on a bad brain day
"It depends" is always a valid answer
The platform never deletes anything without explicit user action
Every tool has an advocacy output mode
Every tool should have an onboarding process that takes into account different types of ND and communication preferences
